# CarHunt

`CarHunt` - iOS-приложение для коллекционирования замеченных автомобилей. Пользователь фотографирует машину, создает карточку, при необходимости заполняет данные вручную или через автозаполнение с бэкенда, сохраняет запись локально и затем просматривает коллекцию в виде сетки или на карте.

## Что Умеет Приложение

Основной сценарий построен вокруг трех вкладок:

1. `Camera` - съемка автомобиля и создание новой карточки.
2. `Collection` - просмотр сохраненных карточек, сортировка, открытие детального превью и удаление.
3. `Map` - отображение машин с координатами и показ фото прямо в `callout` маркера.

## Ключевые Фичи

### 1. Съемка Фото И Создание Карточки

- Захват фото реализован через `AVFoundation`.
- На реальном устройстве приложение запрашивает доступ к камере и показывает live preview.
- После снимка пользователь попадает на экран настройки карточки.
- В симуляторе вместо реальной камеры используется `MockCameraService`.

### 2. Локальная Коллекция Машин

- Карточки хранятся локально через `SwiftData`.
- Экран коллекции показывает карточки в двухколоночной сетке.
- Каждую карточку можно открыть в увеличенном модальном превью.
- Удаление доступно через `context menu`.
- Поддерживается сортировка по оценке, дате, году, марке и мощности.

### 3. Автозаполнение Полей

- На экране настройки карточки есть действие `Autofill`.
- Приложение отправляет фото на бэкенд-эндпоинт `/cards/from-image`.
- Ответ сервиса заполняет:
  - марку
  - модель
  - тип кузова
  - числовую оценку
  - год
  - мощность
  - заметки
- Перед сохранением обязательные поля валидируются, а незаполненные подсвечиваются.

### 4. Отображение Фото На Карте

- Приложение извлекает GPS-метаданные из сделанного фото.
- Координаты сохраняются вместе с карточкой.
- На карте отображаются только карточки, у которых есть координаты.
- При нажатии на маркер открывается `callout`, в котором есть:
  - марка и модель автомобиля
  - дополнительная информация
  - фотография автомобиля

### 5. Оптимизация Изображений

- Сделанные фотографии оптимизируются перед сохранением.
- Base64-изображения декодируются только на границе UI-модели.
- Для списков и карты используется downsampling и кэширование изображений в памяти, чтобы снизить расход ресурсов.

## Технический Стек

- `Swift 5`
- `SwiftUI` для основного интерфейса
- `SwiftData` для локального хранения карточек
- `AVFoundation` для камеры и превью
- `CoreLocation` для геопозиции и GPS-метаданных
- `MapKit` + `UIKit` bridge для карты и кастомных `annotation callout`
- `URLSession` для сетевых запросов
- `ImageIO` и `UniformTypeIdentifiers` для обработки изображений, миниатюр и метаданных
- `CryptoKit` для стабильного преобразования идентификаторов DTO

## Архитектура

Проект организован по feature-модулям и использует `MVVM` на уровне экранов.

### Основные Принципы

- `Features/*` содержат пользовательские сценарии, разделенные на `Views`, `ViewModels` и локальные сервисы.
- `Data/*` содержит локальные модели, DTO, UI-модели и слой доступа к данным.
- `Core/*` содержит сквозную инфраструктуру: роутинг, сеть и общие ошибки.
- Навигация централизована в `AppRouter`.
- Работа с постоянным хранилищем абстрагирована через `CardStorage` и реализована в `CardStorageManager`.

### Поток Данных

1. Пользователь открывает вкладку камеры.
2. `CameraViewModel` делает снимок через `CameraService`.
3. Приложение создает draft-карточку, извлекает GPS-данные и открывает `CardSettingsView`.
4. `CardSettingViewModel` при необходимости вызывает автозаполнение с бэкенда.
5. Готовая карточка сохраняется в `SwiftData`.
6. Сохраненная запись появляется в `Collection` и, если есть координаты, в `Map`.

## Модель Данных

Ключевая сущность проекта - `CardDataModel`. Она хранит:

- уникальный идентификатор
- фото в виде Base64-строки
- марку и модель
- тип кузова
- числовую оценку
- опциональные год, мощность и заметки
- дату добавления
- опциональные широту и долготу

## Запуск И Конфигурация

### Требования

- таргет только для `iPhone` (`TARGETED_DEVICE_FAMILY = 1`)
- минимальная версия: `iOS 17.6`
- Xcode с поддержкой `SwiftUI` и `SwiftData`
